### Selección de Motor de Persistencia

- **Estado:** Aceptado.
- **Contexto:** El sistema requiere persistencia de datos compleja (turnos, reservas, usuarios) sin servidor.
- **Decisión:** Se utilizará **IndexedDB** gestionado a través de la librería **Dexie.js**.
- **Justificación:** A diferencia de `localStorage` (limitado a 5MB y solo strings), IndexedDB permite almacenar objetos complejos, realizar consultas por índices y manejar volúmenes de datos mayores de forma asíncrona. `Dexie.js` facilita la integración reactiva con React mediante `useLiveQuery`.

### Implementación

- **Archivo:** `src/db/database.ts`
- **Tablas (Stores):**
  - `turns`: id, date, startTime, endTime, maxCapacity, status
  - `reservations`: id, turnId, clientName, idCard, reservationDate, status
  - `users`: id, email, password
