
- **RF-01: Visualización de Turnos:** El sistema debe listar turnos disponibles (fecha, hora, cupo) para acceso público.
    
- **RF-02: Reserva de Turnos:** Los clientes deben poder reservar mediante un formulario (nombre y carnet de identidad).
    
- **RF-03: Autenticación de Administrador:** Acceso mediante email y contraseña para gestionar el sistema.
    
- **RF-04: CRUD de Turnos:** El administrador debe poder crear, leer, actualizar y eliminar turnos.
    
- **RF-05: Gestión de Reservas:** El administrador debe poder listar y cancelar reservas (cambio de estado).
    
- **RF-06: Validación de Reglas de Negocio:**
    
    - No permitir solapamientos horarios en la creación de turnos.
        
    - No permitir reservas si el cupo del turno está agotado.
